# 📝 Executive Summary

This talk explores how software engineering is fundamentally shifting from writing code to primarily planning and reviewing AI-generated code. Louis Knight-Webb, founder of Vibe Canvas and AI Tinkers London chapter organizer, argues that as AI coding tools become increasingly capable, the core value engineers provide shifts from implementation to strategic direction and quality assurance. The talk covers the historical evolution from manual coding through GitHub Copilot to autonomous agents, examines time horizons and how longer-running agents change workflow dynamics, and includes a candid discussion about shutting down his startup Vibe Canvas. This represents a significant mental model shift for any software engineer adapting to the AI era.

---

# 🏗️ Core Framework / Concept

## What It Is

> 🏗️ **The "Plan and Review" paradigm** is the emerging model where software engineers spend the majority of their time designing, planning, and reviewing AI-generated code rather than writing it themselves.

This represents a fundamental inversion of the traditional software engineering workflow, where the ratio of planning/reviewing to writing code has flipped dramatically.

## When To Use

> ✅ **Use this when:** Working with AI coding agents (Cursor, Windsurf, GitHub Copilot), building software in environments with high AI adoption, managing AI-augmented development teams

> ❌ **Don't use when:** Working on completely novel problems without existing patterns, when AI context windows are too small for your task, situations requiring deep security auditing

## Key Components

> ⚙️ **Planning:** Defining what to build, breaking down requirements, architecting solutions before AI implementation

> ⚙️ **Reviewing:** Evaluating AI-generated code for correctness, security, performance, and alignment with project goals

> ⚙️ **Agent Management:** Directing and course-correcting AI agents running for extended time horizons

> ⚙️ **Strategic Thinking:** Understanding the "why" behind implementations, not just the "how"

# 📚 Step-by-Step Implementation

## Step 1: Accept the Role Shift

> 🔹 **What to do:** Acknowledge that writing code is no longer your primary value-add. Your value is in direction, review, and quality control.

> 💡 **Why:** AI is objectively better at generating code quickly. Fighting this leads to inefficiency.

## Step 2: Master Planning Skills

> 🔹 **What to do:** Invest heavily in breaking down requirements, writing clear specifications, and architecting solutions before AI implementation begins.

```
Good prompt = Good code
Your job is making good prompts through clear planning
```

## Step 3: Become an Expert Reviewer

> 🔹 **What to do:** Develop strong code review skills focused on AI-generated output—check for logic errors, security issues, and alignment with requirements.

> 💡 **AI writes code fast; you need to review it faster**

## Step 4: Manage Longer Time Horizons

> 🔹 **What to do:** Learn to direct agents that run for hours or days. This requires different skills than short prompt-response cycles.

# 💡 Key Insights (3-5 bullet points)

> 💡 **The ratio has inverted:** Previously 80% writing / 20% planning-review. Now it's reversed with AI assistance.

> 💡 **AI agents are running longer:** Time horizon for AI tasks is extending from minutes to hours/days, requiring new management skills.

> 💡 **Planning quality determines output quality:** "Good prompt = Good code" — the engineer's value moves upstream to specification.

> 💡 **Your company may need to change:** Traditional software company structures may not fit the new AI-first workflow (referencing his startup shutdown).

> 💡 **Review is the bottleneck:** AI can write code faster than humans can review it — this becomes the new constraint.
